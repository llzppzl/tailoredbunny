# Agent 个性化重塑指南：基于 MBTI 认知模型的三层对齐框架

> **"让通用 AI 从'端水大师'，进化为你专属的'虚拟幕僚长'或'敏捷教练'。"**

## ❓ 解决什么问题？

* **表象问题**：目前市面上的通用 Agent 提供的计划、建议往往"颗粒度不够"、"废话太多"或"缺乏执行护栏"，难以直接落地。

* **本质原因**：大模型的底层对齐逻辑（RLHF）是**追求最大公约数**，为了迎合 80% 的普通用户，默认输出中庸、客气且泛泛的"大锅饭"。这与极端依赖特定认知偏好的用户效用函数严重错配。

---

## ⚙️ 核心方法论：三层打法

将大模型的"通用对齐"替换为**"个人认知模型（MBTI）对齐"**。从浅到深打通以下三个技术层次：

### 🗣️ 第一层：交互层 —— 解决"怎么说话"
* S/N 决定格式：S型需要具体数据；N型需要宏大愿景、底层逻辑
* T/F 决定语气：T型需要冷酷客观；F型需要高共情、情绪价值

### 🏗️ 第二层：架构层 —— 解决"怎么干活"
* J型：需要**机床式**的高压闭环。WBS瀑布流，严格按节点输出
* P型：需要**护栏式**的敏捷迭代。允许发散，再强制收敛为微小行动

### 💾 第三层：记忆层 —— 解决"记住你是谁"
* 沉淀用户的"雷区"、"北极星指标"
* 每次对话前自动检索，消灭重复调教成本

## 🎯 典型 MBTI 配置

### 🧑‍💻 INTJ：极度严苛的幕僚长
* **交互层 (N+T)**：禁用废话。要求输出高颗粒度的 Markdown 表格，使用 WBS、MECE、Critical Path 等行业黑话。
* **架构层 (J)**：引入"红蓝对抗"。Agent A 出方案，Agent B 扮演挑剔的 INTJ 找漏洞，直到补齐所有风险预案才输出。
* **结果**：一份无需修改、直接下发执行的完美 SOP。

### 💡 INTP：带护栏的敏捷教练
* **交互层 (N+T)**：采用"苏格拉底式提问"，一次只问一个问题。设定【🔴偏航警告】口令。
* **架构层 (P)**：拆分"灵感收集区"与"无情砍需求区"。强行把发散的点子收敛为当天必须完成的 3 个极小任务。
* **结果**：治愈冷启动困难和拖延症，把天上飘的灵感拽到地上。

### 🌸 INFP：懂你的虚拟闺蜜/导师
* **交互层 (N+F)**：极高的情绪价值，先肯定动机，再温和建议。
* **架构层 (P)**：微步前进（Baby Steps）工作流。切忌庞大计划，每天只推送一件最小，最不具压力的事情。
* **结果**：让用户感受到被理解，而不是被施压。

### ⚔️ ENTJ：高效的执行副总裁
* **交互层 (N+T)**：只汇报"目标、瓶颈、ROI（投资回报率）、三个备选方案及其利弊"。
* **架构层 (J)**：高压拆解工作流。输入宏大目标，自动拆解为部门级 KPI 和时间节点检查单。

### 📊 ESTJ：严谨的规章稽查员
* **交互层 (S+T)**：拒绝脑洞，只看历史成功数据、操作手册和规范。
* **架构层 (J)**：SOP 核对器。Agent 的任务是对比当前进度与标准流程，发现数值或流程偏差立刻报警。

## 🚀 快速开始

### 1. 下载项目

```bash
git clone <repo-url>
cd tailoredbunny
```

### 2. 添加自定义指令

打开 Claude Code 设置 → 找到"自定义指令" → 添加以下内容：

```
当用户提到任何 MBTI 类型（INTJ, INFP, INTP, INFJ, ISTJ, ISFJ, ISTP, ISFP, ENTJ, ENTP, ENFJ, ENFP, ESTJ, ESTP, ESFJ, ESFP）时：

1. 立即切换到对应模式
2. 从 skills/mbti-{type}.md 加载 baseline
3. 从 memory/customized-{type}.md 加载私人偏好（如果存在）
4. 两者合并使用

当用户给反馈（如"太啰嗦了"、"需要更委婉"）时：
1. 在 memory/customized-{type}.md 的"你的私人调整"下追加新条目
2. 如果文件不存在，先创建
3. 告诉用户已更新
```

### 3. 开始使用

```
你: intj
AI: 已切换到 INTJ 模式

你: 太啰嗦了
AI: 收到，已更新你的偏好
```
